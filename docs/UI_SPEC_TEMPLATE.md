# UI Spec Template

## Summary
- Page or feature name
- Goal and success criteria
- Target user and primary action

## Layout
- Route and layout file
- Page structure (sections in order)
- Responsive behavior (mobile, tablet, desktop)

## Content
- Headline and supporting copy
- Data sources (content, server, API)
- Empty states and error states

## Components
- UI primitives to use (Button/Card/Badge/CodeBlock)
- Custom components required (name + purpose)
- Props and variants

## Visual System
- Token usage (colors, gradients, shadows, glow)
- Typography scale (heading levels, body size)
- Spacing scale

## Motion
- Interaction states (hover, focus, active)
- Animation rules (duration, easing)
- Constraints (no custom motion outside tokens)

## Accessibility
- Keyboard behavior
- Contrast requirements
- ARIA considerations

## QA Checklist
- Pixel alignment with tokens
- No hardcoded colors
- No default exports
- Server components by default
